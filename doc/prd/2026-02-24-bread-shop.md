# 🍞 빵가게 (Bread Shop) - PRD

**날짜:** 2026-02-24
**작업명:** 빵가게 (Bread Shop) - 외부 GitHub 데이터셋 동적 로드 및 관리

---

## 1. 개요 (Overview)

현재 '암기빵' 앱은 앱 내부에 포함된 고정된 데이터셋(`assets/datasets/`)만 학습할 수 있습니다. 사용자가 더 다양하고 최신의 학습 데이터를 즐길 수 있도록, 외부 GitHub 저장소를 '빵가게'처럼 활용하여 원하는 학습 데이터(빵)를 브라우징하고 자신의 '주머니(로컬 저장소)'에 담아올 수 있는 기능을 추가합니다.

---

## 2. 주요 개념 (Core Concepts)

### 2.1 빵집의 구조 (Store Structure)
*   **공식 빵집 (Official Repo):** 특정 GitHub 리포지토리(예: `memory-bread/bakery`)를 기본 데이터 소스로 사용합니다.
*   **코너 (Corner):** 리포지토리 내의 **폴더**입니다. (예: `영어단어`, `과학상식` 폴더)
    *   각 코너에는 `corner.json` 파일을 두어 해당 코너의 이름, 설명, 대표 아이콘(Emoji 또는 Image URL)을 설정할 수 있습니다.
*   **빵 (Bread):** 실제 학습 데이터인 **JSON 파일**입니다.
    *   개별 빵 파일에도 메타데이터(빵 이름, 난이도 등)를 포함하거나, 파일명을 빵 이름으로 사용합니다.

### 2.2 구매 및 소유 (Ownership & Sync)
*   **진열대 (Showcase):** GitHub에서 실시간으로 불러온 목록입니다.
*   **구매한 빵 (Purchased):** 로컬 저장소(`SharedPreferences` 또는 앱 내부 저장공간)에 다운로드되어 언제든 학습 가능한 상태입니다.
*   **재료 갱신 (Update):** GitHub의 데이터가 변경되었을 경우(Hash나 날짜 비교), 기존에 다운로드한 빵을 최신 버전으로 덮어쓸 수 있습니다.

## 3. 기술 요구사항 (Technical Requirements)

1.  **GitHub REST API 활용:**
    *   `GET /repos/{owner}/{repo}/contents/{path}`: 특정 경로의 파일 및 폴더 목록 조회.
2.  **데이터 다운로드:**
    *   `raw.githubusercontent.com`을 통해 JSON 데이터 원문 확보.
3.  **상태 관리 및 저장:**
    *   `StorageService`: 로컬 자산(assets)과 다운로드된 자산(user-defined)을 구분하여 통합 관리.
    *   다운로드된 데이터의 원본 URL, 다운로드 시점, 데이터 해시(Etag 등) 저장.
4.  **UI/UX:**
    *   '빵가게' 전용 화면: 코너별 가로 스크롤 또는 그리드 뷰.
    *   다운로드 진행률 표시 및 성공/실패 피드백.

## 4. 개발 단계 (Development Phases)

### Phase 1: 기반 구축 (Foundation)
- [ ] GitHub API 연동 서비스 구현 (`BakeryService`).
- [ ] 특정 리포지토리의 루트 폴더(코너) 목록 가져오기 로직 개발.
- [ ] 단일 JSON 파일 다운로드 및 로컬 저장 기능.

### Phase 2: 빵가게 UI (Bakery UI)
- [ ] `BakeryScreen` (빵가게 메인) 구현: 코너 리스트 표시.
- [ ] `CornerDetailScreen`: 코너 내부의 빵(파일) 목록 표시.
- [ ] 빵별 상태 표시 (새로운 빵, 이미 가진 빵, 업데이트 필요).

### Phase 3: 관리 및 동기화 (Management)
- [ ] 로컬 저장소 확장: 외부에서 가져온 빵들을 `DatasetBrowser`에서 함께 표시.
- [ ] 중복 체크 및 데이터 갱신(Update) 로직 구현.
- [ ] 저장 용량 관리 (다운로드한 빵 삭제 기능).

### Phase 4: 고도화 (Polishing)
- [ ] `corner.json` 기반의 커스텀 테마(색상, 아이콘) 반영.
- [ ] 검색 기능 추가.
- [ ] 오프라인 모드 대응 (이미 다운로드한 빵은 네트워크 없이 목록 노출).

## 5. 연관 아이디어 및 개선사항

*   **나만의 레시피 공유:** 사용자가 직접 자신의 리포지토리 URL을 입력하여 '나만의 빵집'을 추가할 수 있는 기능.
*   **커뮤니티 빵집:** 인기 있는 데이터셋 리포지토리를 추천 리스트로 제공.
*   **빵 굽기 도구:** 앱 내에서 간단하게 JSON 형식을 생성해 주는 '제빵기(Editor)' 기능 연동.
*   **별점 및 후기:** GitHub Stars 또는 별도의 간단한 피드백 시스템(선택 사항).
