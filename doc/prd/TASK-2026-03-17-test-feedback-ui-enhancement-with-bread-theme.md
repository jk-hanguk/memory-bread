# TASK-2026-03-17: 테스트 화면 피드백 UI 개선 (빵 테마 적용)

## 1. 개요
현재 '소화 확인(테스트)' 모드에서 사용자가 답을 선택하면 즉시 다음 문제로 넘어가버려 정답/오답 여부를 명확하고 즐겁게 인지하기 어렵습니다. 이를 개선하기 위해 빵 테마에 어울리는 짧고 직관적인 시각적/촉각적 피드백을 추가합니다.

## 2. 요구 사항 및 가이드라인
- **즉각적이고 짧은 피드백:** 피드백은 사용자의 학습 흐름을 방해하지 않도록 매우 짧아야 합니다 (예: 0.5초 ~ 0.8초). 피드백 후 자동으로 다음 문제로 넘어갑니다.
- **빵 테마 적용:** 정답과 오답에 대해 앱의 테마인 '빵'과 관련된 재미있는 시각적 효과를 제공합니다.
- **다양성:** 피드백이 지루하지 않도록 여러 종류의 효과를 랜덤하게 제공합니다.
- **촉각 피드백:** 시각적 효과와 함께 기기의 진동(Haptic)을 활용하여 타격감을 더합니다.

## 3. 피드백 아이디어 상세

### 3.1 정답 시 피드백
- **버튼 색상 변화:** 선택한 정답 버튼이 초록색 계열로 잠시 변경됩니다.
- **시각적 효과 (랜덤 1종):**
  1. 🍞 한 입 베어 문 빵 이미지 팝업 (또는 애니메이션)
  2. 🥐 크루아상, 🥖 바게트 등 다양한 빵 이모지가 팡 터지는 파티클 효과
  3. 화면 중앙에 크게 "O" 표시와 함께 맛있는 빵 아이콘 표시
- **촉각 피드백:** 가벼운 성공 진동 (`HapticFeedback.lightImpact()`).

### 3.2 오답 시 피드백
- **버튼 색상/동작 변화:** 선택한 오답 버튼이 붉은색 계열로 변경되며 좌우로 흔들리는(Shake) 애니메이션 적용.
- **시각적 효과 (랜덤 1종):**
  1. ⬛ 검게 타버린 빵 이미지 팝업
  2. 🐀 쥐(또는 새)가 빵을 물고 도망가는 아이콘 팝업
  3. 화면 중앙에 크게 "X" 표시와 함께 부서진 빵 아이콘 표시
- **촉각 피드백:** 묵직한 실패 진동 (`HapticFeedback.heavyImpact()`).

## 4. 구현 계획
1. `test_screen.dart` 파일 수정.
2. 답안 선택 시 즉시 넘어가는 로직(`_answerQuestion`)을 수정하여 선택 상태를 저장하고 지연(Delay)을 줍니다.
3. 지연 시간 동안 UI에 피드백 효과(버튼 색상 변경, Shake 애니메이션, 오버레이 아이콘 등)를 렌더링합니다.
4. `flutter/services.dart`의 `HapticFeedback`을 추가합니다.
5. 이모지를 활용한 간편하고 귀여운 랜덤 피드백 위젯을 구현합니다.
